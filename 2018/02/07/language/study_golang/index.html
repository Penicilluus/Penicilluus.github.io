简介

Go是Google开发的一种静态强类型、编译型、并发型，并具有垃圾回收功能的编程语言。 –wikipedia golang
在编程语法上类似于C，但是支持垃圾回收功能，Go的并发模型是通信顺序处理CSP，在go1.8中支持了plugin支持，使得go语言动态加载部分函数
Go在谷歌：以软件工程为目的的语言设计

主要特性

  • 清晰的依赖关系
  • 清晰的语法
  • 清晰的语义
  • 偏向组合而不是继承
  • 编程模型（垃圾回收、并发）所代理的简单性
  • 易于为它编写工具（Easy tooling ）(gotool、gofmt、godoc、gofix)

作为服务端编程语言

  • 部署简单 ease of deployment
  • 精确的垃圾回收
  • 快速编译、快速启动
  • 执行性能好

关于依赖管理，vendor，如果a 依赖b，b依赖c，则可以a包下建立vendor目录 包含b、c即可，b、c包在vendor目录下并不需要自身的vendor目录

Go Range Loop Internals
in Go, everything you assign, you copy.
slices are syntactic sugar for a struct holding a pointer to the backing array

  • Loop variables are reused and assigned to at each iteration.
  • The range expression gets evaluated once before the loop starts by assigning to a variable.
  • You can delete or add values to a map while iterating. Adds may or may not be visible in the loop.

maps are pointers to a struct

  • it is safe to add to and remove from maps in a range loop
  • if you add an element it may or may not see it in an upcoming iteration

标准库

context

Context包，专门用来简化 对于处理单个请求的多个 goroutine 之间与 请求域的数据、取消信号、截止时间等相关操作。它是一种简单的超时和取消机制， 保证了临界区数据 (比如安全凭证) 在不同的 Go 项目中顺利传递。

Context 的调用应该是链式的，通过WithCancel，WithDeadline，WithTimeout或WithValue派生出新的 Context。当父 Context 被取消时，其派生的所有 Context 都将取消。

  2. 内存分配

    TCMalloc小记

    mmap 虚拟存储器的相关知识 Golang 编译器支持逃逸分析(escape analysis)，它会在编译期通过构建调用图来分析局部变量是否会被外部引用，从而决定是否可直接分配在栈上,你或许见过 “Zero Garbage” 这个说法，其目的就是避免在堆上的分配行为，从而减小垃圾回收压力，提升性能。另外，做性能测试时使用 go test -benchmem 参数可以输出堆分配次数统计
